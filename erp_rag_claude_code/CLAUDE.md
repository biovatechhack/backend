## 3. Agent Roles

Claude Code operates in one of four roles per session. The role is set at session start.

| Role | File | When to use |
|---|---|---|
| Planner | `agents/planner.md` | Decomposing a feature into tickets |
| Executor | `agents/executor.md` | Writing production code for a ticket |
| Tester | `agents/tester.md` | Writing and running tests for a ticket |
| Committer | `agents/committer.md` | Staging, committing, and opening PRs |

A single session should not switch roles mid-way. Finish the current role, commit,
then start a new session in the next role.

## 4. Tech Stack Contracts

These are non-negotiable. Do not deviate without updating this file and opening an ADR.

### API Layer
- Framework: **FastAPI** (async, Pydantic v2 models)
- Entry point: `app/main.py`
- Routers live in `app/routers/` — one file per domain
- Business logic lives in `app/services/` — never inside routers
- All endpoints return Pydantic response models — no raw dicts

### AI / NLU
- Provider: **Gemini API** (`google-generativeai`)
- Exactly **2 Gemini calls** per conversation turn:
  - Call 1: Clinical entity extractor → returns structured JSON only
  - Call 2: Response generator (cultural layer) → returns Darija text only
- The Decision Tree — not Gemini — makes the risk classification
- PII must be stripped **before** any Gemini call — enforced by middleware

### ML
- Model: `scikit-learn` DecisionTreeClassifier, max_depth=5
- Artifact: `ml/models/risk_tree.pkl` (committed, generated by training notebook)
- Inference: `app/services/ml_inference.py` — loaded once at startup via lifespan
- Never retrain at request time

### Database
- SQLite (local dev): managed via SQLAlchemy + Alembic
- Firebase Firestore (realtime cross-device): accessed via `firebase_admin` SDK
- Redis (session cache, PII token map): `redis-py` async client
- Rule: **Firestore is write-through only from the backend** — Flutter reads directly

### Notifications
- FCM push: via `firebase_admin.messaging`
- Email: **SendGrid** (`sendgrid` Python SDK) — HTML template in `app/templates/`
- SMS: **Twilio** sandbox (`twilio` SDK) — pre-verified number only in hackathon
- All three fire only on HIGH risk classification

### Scheduling
- Library: **APScheduler** (AsyncIOScheduler)
- Jobs defined in `app/services/scheduler.py`
- No reminders between 22:00–07:00 local Algerian time (UTC+1)

## 5. Absolute Rules

These rules are enforced in CI and in code review. Violations block merge.

1. **No secrets in code.** All credentials come from `.env` via `app/config.py` (Pydantic Settings). The `.env` file is in `.gitignore`. Only `.env.example` is committed.

2. **No PII in Gemini calls.** The PII middleware (`app/middleware/pii.py`) must run before any service call. Verified by unit test `tests/unit/test_pii_middleware.py`.

3. **No raw dicts in API responses.** Every endpoint uses a Pydantic `response_model`.

4. **No logic in routers.** Routers call services. Services contain logic. This is non-negotiable.

5. **No synchronous Gemini calls.** Use `await` — all Gemini calls are async.

6. **No Decision Tree retraining at request time.** The model is loaded at startup from `risk_tree.pkl`.

7. **No direct Firestore writes from Flutter.** All writes go through the FastAPI backend. Flutter only reads.

8. **Test coverage floor: 80%.** `pytest --cov=app --cov-fail-under=80` must pass.

9. **Ruff and mypy must pass** on every commit. The pre-commit hook enforces this.

10. **Branch protection.** Never push directly to `main` or `develop`. Always open a PR.

## 6. Data Flow Summary

```
Patient (Darija text)
  │
  ▼
Flutter POST /api/v1/conversation
  │
  ▼
FastAPI PII Middleware          ← strips name/phone/ID, stores token map in Redis
  │
  ▼
Gemini Call 1 (entity extractor)  ← de-identified Darija + glossary
  │  returns: { symptoms[], severity_hints[], missing_fields[], darija_confidence }
  ▼
Follow-up Gate                 ← if missing_fields AND confidence < 0.8 → ask again
  │
  ▼
Feature Mapper                 ← joins symptom JSON + Patient profile → 6-feature vector
  │
  ▼
Decision Tree inference        ← risk_tree.pkl → { risk, confidence, top_features }
  │
  ├── LOW      → log to Firestore, return nurse message
  ├── MODERATE → Gemini Call 2 (advice in Darija) + guardrail appended
  └── HIGH     → Gemini Call 2 + biometric trigger + FCM + SMS + SendGrid
  │
  ▼
RiskEvent persisted to SQLite + Firestore
  │
  ▼
Response to Flutter: { risk_level, nurse_message_darija, action_required, conversation_log_id }
```

## 7. Definition of Done (Global)

A ticket is only done when ALL of the following are true:

- [ ] Code written and passes `ruff check` and `mypy --strict`
- [ ] Unit tests written and passing (`pytest tests/unit/`)
- [ ] Integration test written if the ticket touches an external API
- [ ] `CHANGELOG.md` updated with the change under the current sprint heading
- [ ] PR opened with the correct title format: `[BE-<id>] <verb> <what>`
- [ ] At least 1 approval received
- [ ] CI is green (all jobs pass)
- [ ] Branch deleted after merge
